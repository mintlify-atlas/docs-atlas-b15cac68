## What is Gestión Gente?

Gestión Gente is a full-stack HR management platform built for Peruvian businesses operating across multiple branches (sedes). It centralizes employee lifecycle management, real-time attendance tracking, vacation planning, and organizational configuration behind a role-based access control system.

The platform is built with React, Vite, and Tailwind CSS on the frontend, and uses Supabase as its backend — providing a PostgreSQL database, authentication, Row Level Security policies, and real-time subscriptions out of the box.

## Platform architecture

Gestión Gente uses a single-page application architecture deployed at `/sebi-web`:

- **Frontend**: React 18 with Vite, Tailwind CSS v4, React Router v6
- **Backend**: Supabase (PostgreSQL, Auth, Realtime, Storage)
- **Authentication**: Supabase Auth with email/password, session management via `AuthContext`
- **Authorization**: RBAC via `role_modules` table and Row Level Security policies
- **Charts**: Recharts for data visualization
- **Excel**: `@e965/xlsx` for import/export operations
- **PDF**: `html2pdf.js` for generating vacation vouchers (papeletas)

## User roles

The platform supports a hierarchy of user roles, each with configurable module permissions:

| Role | Description |
|------|-------------|
| **Super Admin** | Full system access, set via Supabase `app_metadata` |
| **Admin / Jefe de Gente** | Global access to all employees and data |
| **Analista de Gente** | HR analyst at ADM. Central with broad read access |
| **Gerente / Jefe de Área** | Manages employees within their area across sedes |
| **Supervisor / Coordinador** | Manages their direct team at a specific sede |
| **Employee** | Limited access to their own data |

<Note>
  User roles and their module permissions are fully configurable from the **Roles Management** page. The role hierarchy described above reflects the default configuration shipped with the platform.
</Note>
