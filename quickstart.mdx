This guide walks you through cloning the project, configuring environment variables, and signing in for the first time.

## Prerequisites

<CardGroup cols={3}>
  <Card title="Node.js" icon="node-js">
    Version 18 or later. Download from [nodejs.org](https://nodejs.org).
  </Card>
  <Card title="npm" icon="npm">
    Included with Node.js. Used to install dependencies and run scripts.
  </Card>
  <Card title="Supabase account" icon="database">
    The platform uses a self-hosted Supabase instance at `supabase.pauserdistribucionessac.com`.
  </Card>
</CardGroup>

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone the project and move into the directory:

    ```bash
    git clone https://github.com/Pauser-Devops/web_gestiongente.git
    cd web_gestiongente
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages:

    ```bash
    npm install
    ```

    Key runtime dependencies include:

    | Package | Version | Purpose |
    |---|---|---|
    | `@supabase/supabase-js` | ^2.90.1 | Supabase client |
    | `react` | ^18.3.1 | UI framework |
    | `react-router-dom` | ^6.22.3 | Client-side routing |
    | `recharts` | ^2.12.3 | Charts and analytics |
    | `lucide-react` | ^0.562.0 | Icon library |
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.development` file in the project root. The dev server reads these variables to connect to the Supabase backend through a local proxy.

    ```bash .env.development
    VITE_SUPABASE_URL=http://localhost:5173/supabase-proxy
    VITE_SUPABASE_ANON_KEY=<your-anon-key>
    ```

    <Note>
      During local development, `VITE_SUPABASE_URL` points to the Vite dev server's built-in proxy (`/supabase-proxy`), which forwards requests to `https://supabase.pauserdistribucionessac.com`. You do not need to change this value for local development.
    </Note>

    <Warning>
      Never commit `.env.development` to version control. It contains the Supabase anon key, which grants public API access. Add it to `.gitignore` if it is not already there.
    </Warning>

    The Vite proxy is configured in `vite.config.js`:

    ```javascript vite.config.js
    server: {
      proxy: {
        '/supabase-proxy': {
          target: 'https://supabase.pauserdistribucionessac.com',
          changeOrigin: true,
          rewrite: (path) => path.replace(/^\/supabase-proxy/, ''),
          secure: true,
        },
      },
    },
    ```
  </Step>

  <Step title="Start the development server">
    Run the dev server:

    ```bash
    npm run dev
    ```

    Vite starts a local server (typically at `http://localhost:5173/sebi-web/`). The app is served under the `/sebi-web/` base path as configured in `vite.config.js`.

    <Tip>
      The other available scripts are `npm run build` (production bundle), `npm run preview` (preview the build locally), and `npm run lint` (run ESLint).
    </Tip>
  </Step>

  <Step title="Log in">
    Open `http://localhost:5173/sebi-web/login` in your browser.

    Enter your corporate email address and password, then click **INGRESAR**.

    The login page validates your credentials against Supabase Auth (email/password). After a successful sign-in, the app loads your employee profile and redirects you to the dashboard.

    <Note>
      Your password must be at least 6 characters. If your email has not been confirmed in Supabase Auth, you will see a prompt to verify your inbox before proceeding.
    </Note>
  </Step>

  <Step title="Explore the dashboard">
    After logging in you land on the main dashboard at `/`. What you see depends on your role and the modules your role has access to. Common sections visible after login include:

    - **Dashboard** — summary metrics and key indicators
    - **Employees** — employee directory and registration
    - **Attendance** — attendance records and validation
    - **Requests** — leave and vacation requests
    - **Reports** — exportable HR reports

    <Note>
      If your role does not have `web_access` enabled, or your employee record is marked inactive, the platform will sign you out automatically and show an error message. Contact your HR administrator to resolve access issues.
    </Note>
  </Step>
</Steps>
